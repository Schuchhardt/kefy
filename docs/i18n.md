# Cómo agregar un nuevo idioma a Kefy

## Estructura del sistema i18n

```
locales/
  types.ts          ← Interfaces TypeScript (no tocar al agregar idioma)
  es/
    landing.ts      ← Copy de landing page
    common.ts       ← Metadata, notFound, keywords
    blog.ts         ← Labels de sección blog
    dashboard/
      home.ts
      analytics.ts
      brand.ts
      strategy.ts
      calendar.ts
      settings.ts
      autopilot.ts
      ads.ts
      content.ts
      engage.ts
      inbox.ts
  en/
    (misma estructura)
lib/i18n.ts         ← Array de locales activos
```

---

## Pasos para agregar un idioma (ejemplo: Portugués `pt`)

### 1. Registrar el locale

En `lib/i18n.ts`:

```diff
-export const locales = ['es', 'en'] as const;
+export const locales = ['es', 'en', 'pt'] as const;
```

El tipo `Locale` se actualiza automáticamente por inferencia.

---

### 2. Crear los archivos de locale

Crear la carpeta `locales/pt/` con todos los archivos. Deben satisfacer los tipos de `locales/types.ts`.

**Archivos a crear:**

| Archivo | Tipo exportado |
|---------|----------------|
| `locales/pt/landing.ts` | `KefyCopy` (default export) |
| `locales/pt/common.ts` | `CommonCopy` (default export) |
| `locales/pt/blog.ts` | `BlogCopy` (default export) |
| `locales/pt/dashboard/home.ts` | objeto inferido (default export) |
| `locales/pt/dashboard/analytics.ts` | objeto inferido |
| `locales/pt/dashboard/brand.ts` | objeto inferido |
| `locales/pt/dashboard/strategy.ts` | objeto inferido |
| `locales/pt/dashboard/calendar.ts` | objeto inferido |
| `locales/pt/dashboard/settings.ts` | objeto inferido |
| `locales/pt/dashboard/autopilot.ts` | objeto inferido |
| `locales/pt/dashboard/ads.ts` | objeto inferido |
| `locales/pt/dashboard/content.ts` | objeto inferido |
| `locales/pt/dashboard/engage.ts` | objeto inferido |
| `locales/pt/dashboard/inbox.ts` | objeto inferido |

La forma más rápida es copiar los archivos de `locales/en/` y traducirlos.

**Archivos con funciones** (traducir las cadenas dentro, no la firma):

- `locales/pt/dashboard/brand.ts` → `stepOf: (c, t) => \`Passo ${c} de ${t}\``
- `locales/pt/dashboard/autopilot.ts` → `runSuccess: (n) => \`✓ Executado: ${n} publicação(ões) gerada(s)\``
- `locales/pt/dashboard/ads.ts` → `durationLabel`, `boostPay`, `timeAgo`
- `locales/pt/dashboard/content.ts` → `generateType`, `slidesCount`, `scenesCount`
- `locales/pt/dashboard/engage.ts` → `timeAgo`
- `locales/pt/dashboard/inbox.ts` → `timeAgo`

---

### 3. Registrar en las páginas consumidoras

Las páginas de dashboard importan individualmente. Editar cada una:

```diff
// app/[lang]/dashboard/home/page.tsx (ejemplo)
 import esT from '@/locales/es/dashboard/home';
 import enT from '@/locales/en/dashboard/home';
+import ptT from '@/locales/pt/dashboard/home';

-const T = { es: esT, en: enT } as const;
+const T = { es: esT, en: enT, pt: ptT } as const;
```

**Páginas a actualizar** (11 en total):

```
app/[lang]/dashboard/home/page.tsx
app/[lang]/dashboard/analytics/page.tsx
app/[lang]/dashboard/brand/page.tsx
app/[lang]/dashboard/strategy/page.tsx
app/[lang]/dashboard/calendar/page.tsx
app/[lang]/dashboard/settings/page.tsx
app/[lang]/dashboard/autopilot/page.tsx
app/[lang]/dashboard/ads/page.tsx
app/[lang]/dashboard/content/page.tsx
app/[lang]/dashboard/engage/page.tsx
app/[lang]/dashboard/inbox/page.tsx
```

En cada una, la línea de consumo `const t = T[(lang as Locale) ?? 'es'] ?? T.es` no requiere cambios.

**Páginas de blog y layout** (importan ambos idiomas explícitamente):

```diff
// app/[lang]/blog/page.tsx
 import esBlog from '@/locales/es/blog';
 import enBlog from '@/locales/en/blog';
+import ptBlog from '@/locales/pt/blog';

-const blogLabels: Record<string, typeof esBlog> = { es: esBlog, en: enBlog };
+const blogLabels: Record<string, typeof esBlog> = { es: esBlog, en: enBlog, pt: ptBlog };
```

Mismo patrón en `app/[lang]/blog/[slug]/page.tsx`, `app/[lang]/not-found.tsx`, y `app/[lang]/layout.tsx`.

---

### 4. Migraciones de base de datos

#### `kefy_blog_posts` — **requiere migración**

La columna `lang` tiene un CHECK constraint hardcodeado:

```sql
-- 20260514000002_create_blog_posts.sql
lang TEXT NOT NULL CHECK (lang IN ('es', 'en'))
```

Crear una migración en `db/migrations/`:

```sql
-- db/migrations/YYYYMMDDNNNNNN_add_pt_locale.sql

ALTER TABLE kefy_blog_posts
  DROP CONSTRAINT IF EXISTS kefy_blog_posts_lang_check;

ALTER TABLE kefy_blog_posts
  ADD CONSTRAINT kefy_blog_posts_lang_check
    CHECK (lang IN ('es', 'en', 'pt'));
```

Nombrar con el formato `YYYYMMDDNNNNNN_descripcion.sql`.

#### `kefy_brand_kit` — **no requiere migración**

La columna `language` no tiene CHECK constraint, acepta cualquier valor TEXT.

#### Otras tablas — **no requieren migración**

El resto del esquema no tiene restricciones de idioma.

---

### 5. Verificar

```bash
npm run build
```

TypeScript validará automáticamente que todos los archivos de locale satisfacen las interfaces de `locales/types.ts`. Si algún campo falta o tiene tipo incorrecto, el build fallará con un error descriptivo.

| `app/[lang]/dashboard/*/page.tsx` | Agregar import + entrada en `T` (11 archivos) |

| `app/[lang]/blog/[slug]/page.tsx` | Agregar import + entrada en `labels` |

| `app/[lang]/layout.tsx` | Agregar import + entrada en `commonCopy` |
